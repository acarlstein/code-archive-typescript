Patterns: Creational: Abstract Factory

In Construction - Ignore this page

Definition
The purpose is to provide a generic interface for the creation of concrete objects that are part of a common theme.
This pattern relies on object composition and allow to separate the details of implementation of a set of object from theri general usage.

Usages

  • When you have a family of dependent or related objects. For example: Cars, Trucks, Motorcycles belong to a family of vehicles.
  • Allows for the encapsulation of dependencies and makes an application easier to be portable.
  • The client doesn't required to use the new operator.
  • Allows for minimal changes if a change of platform is required.

Components

  • Abstract Factory class which is public.
  • Factory implementation which is protected.
  • Interfaces which are public.
  • Set of implementation for the objects which is protected.
